十、小技巧

  1. 多用 git status：不确定状态时，先看看
  2. commit 信息要写清楚：feat:、fix:、docs: 这样开头，方便看
  3. 分支命名规范：feature/xxx、bugfix/xxx、hotfix/xxx
  4. 不会就 git status：它会告诉你下一步该干啥

十一、血泪教训

  1. 不要在 main 分支上直接开发：先建分支！
  2. push 之前先 pull：不然容易冲突
  3. 不要用 git push --force：除非你真的知道自己在干嘛
  4. commit 前看看 git diff：确认改了啥再提交
