文件名 作用 详细说明
main.m 整个程序的主入口（最重要） 负责读取所有 .obj 模型、转换贝塞尔曲面、设置动画逻辑、调用 visualizePart 绘制僵尸和豌豆射手、控制动画循环

convertQuadToBezier.m 四边形转 Bézier 曲面 把僵尸模型的四边形面片转换为 4×4 控制点的 Bézier 曲面片（核心曲面重建函数）

discretizeBezier.m Bézier 曲面离散化（生成网格） 把 Bézier 控制点离散成可以画的顶点和四边形面片
discretizeBezierTri.m 三角形 Bézier 离散化 处理三角形 Bézier 曲面的离散
genEdgePoints.m / genBezierEdgePoints.m 生成边上的控制点 为相邻面片的共享边生成中间控制点，保证拼接处 G¹ 连续

readQuadObj.m 读取 .obj 文件 读取僵尸的顶点、面片（支持四边形和三角形）

create_and_move_ball.m 生成并移动豌豆球 画一个绿色球并让它沿 X 轴移动（发射动画）
hat_move.m / move_hat.m 帽子移动函数 用于帽子掉落或移动的动画（未在主程序中使用）

怎么去搭建一个适合自己的知识库？

发表于 2026-03-22 更新于 2026-03-23 分类于 方法

写在前面

一直想着搭建一个自己的知识库，想着学习的知识不只是输入，不只是为了单纯的应付考试，因为这样我觉得就完全失去了学习的意义，这样的学习显得那么的索然无味和单调。

那么问题来了，怎么去搭建一个属于自己的知识库呢？

要解决的问题

结合和ai的对话和自己的思考，得出可以从以下几个方面来进行搭建，知识库的目的是：既保证知识对自己个人能力有提升，也同时打牢考试的地基。

如何去解决

  1. 知识：这一部分主要放“是什么”，总之，只要是你不懂的，很高深的概念都可以放在里面，分为

    • 概念 Wikipedia上搜一搜
    • 解释
    • 案例
    • 延伸

    标题可以写“某某知识到底是什么”、“某某知识有什么实际价值？”

  2. 方法：涉及到解决具体问题的具体方法，这里的结构可以是：

    • 要解决的问题
    • 怎么去解决问题
    • 问题解决方案的思考和拓展

    标题可以是“什么什么方案”、“使用什么什么干了什么事情”

  3. 项目是一个更为宏大的概念，通常会解决一系列的问题，涉及到很多方面的知识

    结构可以是：

    • 为什么要做这个项目

    • 项目运用了哪些方法（对上一级的承接）

    • 实现的过程记录

    • 项目的复盘和反思（这一步重要）

  4. 思考：可以是有关于学习方法的思考，可以是对时代问题的反思，可以是对于个人的总结，总之很自由这部分，天马行空的都行。

  5. 考试：这一部分目的很简单，就是为了考试！考试！考试！

    结构很公式化：

    • 知识点
    • 题型
    • 如何解题
    • 一堆题练习

    怎么说呢，中国的学生，最乐意做的就是这一部分，最擅长的，也是这一部分。

思考

其实这篇文章就是一次对于知识库搭建里“方法”理论的实践，知识库的搭建理论是需要不断去丰富的，我们可以在后面的使用中去不断发现问题，不断优化方法。

这样，才能搭建最适合我们自己的知识库。
