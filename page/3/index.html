GTX1660调试欧卡2画质mod的感触

发表于 2026-03-08 更新于 2026-03-09 分类于 游戏

前期排查

今天在折腾欧卡2的画质。和很多玩家一样，一开始就是一个思路：把能开的都开到最高。包括Nvidia app里面推荐的设置都是都是全开最高，我的gtx1660还怕一个12年的老游戏？

很久之前，我就已经在本地下载了画质类的mod，像 JBX 3、PNG、Realistic Rain、Tree Improved 4K ，今天一时兴起，想玩一下很久没有碰的欧卡2，想着调一下画质，发现，无论是加入PNG，还是JBX 3，一进游戏画质是变好看的，但是帧率低的可怜，估计只有10帧左右，体验极差,这我还玩个寂寞啊…，还好，多亏了我之前看的乐观学书籍，没问题归结为长期性的、无法解决的，不然我估计不会去寻找新的解决方法，而是在寝室怪罪自己这无能的显卡了（泪目）。

自己想了一下，问题可能有很多处：

1.下载的PNG/JBX3画质mod毕竟不是官方的，可能存在问题？

2.画质类mod之间可能存在一定的冲突，可能存在不兼容？

3.最近显卡出了点毛病？

对于1，找了半天发现，原来这些画质类的mod都是要付费的。算了，mod来源不明就这样吧，也挺好，毕竟mod都是免费的，哈哈哈（笑）

对于2，我尝试把Realistic Rain、Tree Improved 4K这些比较吃显卡性能的mod都删掉，再次重新进游戏，发现然而并没有什么用，画质依旧10帧（哭了）

对于3，显卡出了毛病吗？没有啊，昨天晚上打吃鸡还好好的呢，睡了一晚，显卡就废了难道？不可能不可能，绝对不可能

还能怎么办，问问GPT老祖，死马当活马医呗，这句话，一语道破天机

GPT说：

  • 1080P + 100% = 正常渲染
  • 200% ≈ 接近 4K
  • 300% 已经接近 6K 渲染

后来检查了一下设置才发现，缩放确实是开到了 300% ，但我想着，这有什么关系？我之前开300%不是照样玩的很顺？

但并没有意识到的是，之前的300%是没有打PNG/JBX3这种大型画质mod

死马当活马医，改一改缩放率呗，试一试总是好的咯

调整之后

把思路改成适合自己配置 > 盲目堆画质，之后，发现打开了新世界的大门

主要改动：

  • 缩放：300% → 100%（260309：缩放其实调整到200%都是可以的，100%还是有点太便宜显卡了）

  • 设置旁边有 菱形的，在不影响太多观感的前提下，能开多低开多低

  • 一些轻量MOD（灯光、雨滴、内饰、地图）尽量不去动他

  • 260309：抗锯齿可以选择 自适应清晰度，选择了SMAA,SMAA+TAA,自适应清晰度 后发现，

    自适应清晰度很真实，在静止的时候很清楚，运动的时候会有一定的运动模糊

帧率立刻稳定了很多，高速基本很顺，稳定在55帧左右，城市也不会明显卡顿。

很多网上的“画质教程”其实默认的是高端显卡配置。如果照搬，很容易变成：

Notability+Alist+天翼云盘 实现笔记自动备份

发表于 2026-03-03 更新于 2026-03-09 分类于 方法

背景

Notability 是我日常使用的笔记工具，但它的 iCloud 同步局限性比较大。iCloud免费空间太少，只有5GB。为了更灵活地管理和长期保存笔记，我选择了 AList 挂载天翼云盘，再通过 Notability 的 WebDAV 自动备份功能，实现笔记的自动同步

准备工作

Notability：支持 WebDAV 自动备份

AList：轻量文件列表程序，可挂载多种存储后端

天翼云盘：作为最终存储空间

本地电脑：运行 AList，提供 WebDAV 服务

步骤一：在本地电脑部署 AList

  1. 下载 AList

    https://github.com/AlistGo/alist

    注意windows电脑下载的是amd64，而不是arm64（已经被坑过一次了哈哈）

  2. 解压alist 程序压缩包，找到你解压出的文件包的位置，输入cmd，按照下图指示操作

    cmd终端输入alist server即可启动alist客户端

  3. 打开浏览器访问 http://localhost:5244

    登录后台管理界面，初始密码在上图中是有的,具体是在"initial password is"后面

步骤二：挂载天翼云盘

  1. 在后台选择 存储 → 添加存储。

  2. 类型选择 天翼云盘。

  3. 填写参数：用户名：天翼云盘手机号
    密码：登录密码
    根文件夹 ID：在天翼云盘网页端打开某个文件夹，URL 最后的数字就是 ID
    Cookie（可选）：如果遇到验证码问题，可以复制浏览器 Cookie

  4. 保存后，AList 前端即可访问天翼云盘文件。

这里的操作不再赘述，alist官方的操作文档喂饭级别的教程很不错了

https://alistgo.com/zh/guide/drivers/189.html

步骤三：配置 Notability 自动备份

  1. 打开 Notability → 设置 → 自动备份。

  2. 选择 WebDAV。

  3. 填写信息：服务器地址：http://<你的电脑IP>:5244/dav/天翼云盘
    用户名/密码：AList 初始化账号密码（如果你更改过密码的话，输入更改后的密码）

    电脑ip使用ipconfig获取，/天翼云盘 必须有，同理，如果你挂载的是百度网盘，相应的路径要更改为 /百度网盘

  4. 保存后，Notability 会自动将笔记备份到天翼云盘。

    （可选，不强求）步骤四：下载Raidrive，本地查看自动备份文件

    1. 安装好raidrive之后

    2. 打开自己的文件管理器，看看本地是不是多出来一个webdav的网络位置，如果有的话就是成功了，可以实时看自己本地的文件了

总结

通过 Notability 内置 WebDAV → AList → 天翼云盘，我实现了一个自动化、轻量化的笔记备份方案，不再需要手动导出，笔记实时备份。

注意事项：确保电脑运行 AList 程序，alist的cmd不要手痒关掉，否则 WebDAV 服务不可用

后续备份：同样，可以把文件拷一份到移动硬盘中，遵循3-2-1备份原则

方案可扩展，AList 还能挂载其他云盘

如果电脑ip地址更改了需要修改webdav地址的,需要实时进行电脑ip地址的获取，后续可以考虑内网穿透实现直接远程访问的方案

单片机和计算机相比，是一个袖珍版计算机。

但是成本低、体积小、结构简单，学习使用单片机是连接计算机原理和结构的最佳选择。

8051单片机内核，所以叫51单片机

RAM类似于电脑内存，会丢失

ROM类似于电脑存储

以为过程会有点难，没想到其实开始很简单

首先，需要USB线把电脑和单片机连接起来

win11电脑系统不需要额外安装驱动，可以直接使用

遇到的一个问题：显示正在检测目标单片机

可以从以下角度来排查一下：

  1. 芯片型号要选对

  2. 冷启动(reset)

  3. COM串口不要选错

    查看步骤：

    1.点击win徽标键

    2.搜索“设备管理器”

    3.获取端口信息（可以读出是COM7）

    4.在stc-isp中选择COM7串口

  4. RX TX帽子盖上
