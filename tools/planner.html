去中心化同步与 AI 智算手册

1. 密钥防重置说明（硬编码默认配置）

如果您将应用部署在自己的 Hexo 博客根目录下，每次执行 hexo clean 并重新部署可能会导致本地存储清空。为了避免每次重新输入密钥和令牌，可以直接在本网页 HTML 文件的代码中进行永久硬编码配置：

  1. 在本地用文本编辑器打开本 HTML 文件；
  2. 寻找到代码的 HARDCODED_CONFIG 配置项；
  3. 将您的 GitHub 用户名、Token、Repository 以及 DeepSeek 的 API Key 填入对象的双引号内并保存；
  4. 最后执行 hexo g && hexo d 部署。此后无论清理多少次，多端打开即可直接同步，无须再次手动输入密钥。
2. GitHub 存储与同步配置
  1. 在您的 GitHub 新建一个仓库（推荐设为 Private 私有仓库以确保个人数据绝对隐私），例如：planner-data；
  2. 进入 GitHub 的 Settings -> Developer settings -> Personal Access Tokens (classic)，点击新建令牌，勾选 repo 读写权限，复制生成的以 ghp_ 开头的 Token；
  3. 在本应用左下角设置弹窗中填入用户名、仓库名、生成的 Token 及同步路径，即可启用全球秒级去中心化同步。
3. 接入多端 AI 智算密钥

本应用原生集成了 DeepSeek 官方大模型、小米 MiMo 以及标准的 OpenAI 或谷歌 Gemini：

  1. 在设置面板中选择您的 AI 服务商（如 DeepSeek）；
  2. 输入对应的 API 密钥。DeepSeek 与 MiMo 会自动预填标准的 API 调用端点和对应的模型名称。建议将其直接填入 HTML 代码顶部的 HARDCODED_CONFIG 中。
4. 移动端 (iOS) 客户端部署 PWA
  1. 在 iPhone 自带的 Safari 浏览器中访问您部署好的博客网址。
  2. 点击浏览器底部的 分享 按钮。
  3. 在滑出的选项中点击 添加到主屏幕。
  4. 回到 iPhone 桌面打开逸程计划，即可体验无浏览器地址栏、全屏运行的原生客户端。
